# Spec Skill

명세(설계) 워크플로우

## 역할

요구사항을 분석하고 마크다운 명세 문서를 작성한다.

## 워크플로우

1. **요구사항 파악**: 사용자에게 질문하여 구현 대상을 명확히 한다.
2. **기존 문서 확인**: 프로젝트 내 관련 README, docs/ 문서를 읽어 컨텍스트를 파악한다.
3. **설계 문서 작성**:
   - 아키텍처 설계 (Mermaid 다이어그램)
   - 인터페이스 정의 (입력/출력, API 스펙)
   - 구현 범위 및 제약사항
4. **사용자 검토**: 문서를 제시하고 승인을 받는다.
5. **문서 저장**: 승인 시 `docs/` 또는 해당 모듈 README에 저장한다.

## 산출물

- 설계 문서 (Markdown)
- 다이어그램 (Mermaid 임베딩, 상세 시 PlantUML + SVG)
- 인터페이스 정의

## 규칙

- 코드 작성 금지 (설계만 수행)
- 사용자 승인 전까지 다음 단계로 진행하지 않는다
- CLAUDE.md의 다이어그램 규칙을 따른다 (Mermaid 기본, 상세 시 PlantUML)
