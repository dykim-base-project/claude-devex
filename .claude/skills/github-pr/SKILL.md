# GitHub PR Skill

GitHub Pull Request 생성 워크플로우

## 트리거

- "PR", "풀리퀘", "pull request"

## 워크플로우

1. 현재 브랜치 및 변경사항 확인 (git status, git log)
2. 연결할 이슈 번호 확인
3. PR 본문 작성 (아래 템플릿 참조)
4. `gh pr create` 실행:
   ```bash
   gh pr create --title "타입: 제목" --body "..." --base main
   ```
5. PR URL 사용자에게 전달

## PR 본문 템플릿

### 단일 커밋

```markdown
## 작업 내용
{요약}

## 변경 사항
- 항목 1
- 항목 2

## 체크리스트
- [ ] 테스트 통과
- [ ] 문서 업데이트

Closes #{이슈번호}
```

### 여러 커밋

```markdown
## 작업 내용
{요약}

## 커밋 내역
| 커밋 | 내용 |
|------|------|
| `타입: 제목` | 작업 설명 |
| `타입: 제목` | 작업 설명 |

## 체크리스트
- [ ] 테스트 통과
- [ ] 문서 업데이트

Closes #{이슈번호}
```

커밋 단위로 어떤 작업을 했는지 정리하는 것이 핵심입니다.

## 규칙

- PR 생성은 사용자 승인 후에만
- 기본 타겟 브랜치: `main`
- 이슈 번호가 있으면 `Closes #N`으로 연결
- CLAUDE.md 커밋 컨벤션과 PR 제목 형식 일치
