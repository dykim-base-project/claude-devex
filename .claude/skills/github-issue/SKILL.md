# GitHub Issue Skill

GitHub 이슈 생성 워크플로우

## 트리거

- "이슈", "issue"

## 워크플로우

1. 이슈 제목 및 내용 파악 (사용자에게 질문)
2. 라벨 자동 매핑:
   | 키워드 | 라벨 |
   |--------|------|
   | 기능, feat | `feat` |
   | 버그, fix | `fix` |
   | 문서, docs | `docs` |
   | 리팩토링 | `refactor` |
   | 기타 | `chore` |
3. 이슈 본문 작성:
   ```
   ## 작업 내용
   {요약}

   ## 체크리스트
   - [ ] 항목 1
   - [ ] 항목 2
   ```
4. `gh issue create` 실행
5. 이슈 번호 기반 브랜치명 제안:
   - `feat/#{번호}-설명` (기능)
   - `fix/#{번호}-설명` (버그)
   - `docs/#{번호}-설명` (문서)

## 규칙

- 이슈 생성은 사용자 승인 후에만
- 라벨이 없으면 자동 생성
- CLAUDE.md 커밋 타입과 라벨 일치
